## 새로운 문제: 드리프트율 공식의 초기 점수 의존성

현재 공식 `(초기 - 최솟값) / 초기`는 두 에이전트의 초기 점수가 다를 때 공정한 비교가 어렵습니다. StateDoc이 초기에 더 좋은 코드를 생성할수록 오히려 불리해지는 구조입니다.

**개선 방향 2가지:**

**방향 A — 절대 하락량으로 비교**
```
드리프트 = 초기점수 - 최솟값  (비율 아닌 절댓값)
```
두 에이전트가 동일한 9.0→9.0 구간이면 똑같이 0점 하락으로 처리됩니다.

**방향 B — 정규화된 점수 기반 비교**
```
초기 점수를 10점으로 정규화 후 이후 점수를 상대적으로 비교
```
초기 점수 차이를 제거하고 수정 과정의 변동만 측정합니다.
