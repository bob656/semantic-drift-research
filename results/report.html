핵심 발견 및 결론

발견 1: 문서/요약 기반 접근법은 Baseline을 능가하지 못한다

StateDoc, CoTDoc-LLM, CoTDoc-AST 모두 Baseline 대비 높은 드리프트를 보였다. 세 버전에 걸쳐 평균 드리프트가 지속적으로 Baseline보다 2~5.4점 높았다.

발견 2: LLM 생성 요약은 hallucination으로 추가 오류를 유발한다

StateDoc과 CoTDoc-LLM에서 LLM이 요약을 생성할 때 데이터 타입(List[Item] → 튜플)을 잘못 해석하거나, 존재하지 않는 메서드를 생성하는 현상이 관찰됐다.

발견 3: 오류 누적 (Error Propagation)

Recursive Summary 방식에서 이전 단계의 잘못된 코드 구조가 요약에 반영되면, 이후 단계도 같은 오류를 상속한다. CoTDoc-AST의 step4 조기 실패가 step5~8까지 연쇄된다.

발견 4: 실제 코드 컨텍스트(Baseline)가 가장 효과적인 메모리 표현

어떠한 추상화나 요약도 실제 코드 자체보다 정확한 정보를 전달하지 못했다. Baseline이 코드를 300자로 잘라도 직접 코드를 보는 것이 LLM 해석 요약보다 신뢰도가 높다.

한계: 모델 의존성

gemma3:12b에서 반복적인 f-string 절단·docstring 미완성 등 코드 생성 불안정이 실험을 오염시킨다. qwen2.5-coder:14b 또는 deepseek-coder-v2:16b 등 코드 특화 모델로 재실험 시 다른 결과가 나올 수 있다.
