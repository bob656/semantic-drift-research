## 1. 연구 배경 및 문제 정의

최근 LLM 기반 코딩 에이전트(예: GitHub Copilot, SWE-agent)는 단순 코드 생성을 넘어 **반복적 수정(multi-turn modification)** 작업을 수행합니다. 개발자가 "기능 A를 추가해줘 → 이제 B도 추가해줘 → C도 바꿔줘"와 같이 단계적으로 요청할 때, LLM은 이전 단계에서 설계한 동작 규칙(행동 계약, behavioral contract)을 후반 단계에서 망각하는 현상이 관찰됩니다.

이 현상을 본 연구에서는 **의미적 드리프트(Semantic Drift)** 라고 정의합니다.

**연구 질문**: 멀티턴 LLM 코드 생성에서 어떤 유형의 행동 계약이, 왜 드리프트하는가?

---

## 2. 기존 연구와의 차별점

| 기존 연구 | 한계 | 본 연구 |
|---|---|---|
| Laban et al. (2505.06120) | 멀티턴 하락을 단일 수치(39%)로만 측정 | 계약 유형별로 분해하여 측정 |
| SWE-agent (하네스 피드백) | 드리프트를 사후 교정 — 원인 분석 없음 | 드리프트 발생 메커니즘 분석 |
| CodeMEM (2601.02868) | 구조적 메모리 효과만 측정 | 어떤 계약이 메모리 없이 사라지는지 분류 |

**핵심 차별점**: 기존 연구는 드리프트를 단일 현상으로 다루지만, 본 연구는 이를 두 가지로 분리합니다.

- **능력 실패(Capability Failure)**: 모델이 복잡한 요구사항을 구현하지 못하는 것
- **계약 드리프트(Contract Drift)**: 이전에 올바르게 구현했던 계약을 후반 단계에서 위반하는 것

이 구분 자체가 기존 평가 방식에 없는 새로운 기여입니다.

---

## 3. 행동 계약 유형 분류 (이론적 기여)

본 연구는 코드 내 행동 계약을 4가지로 분류하고 취약성 순서를 제안합니다.

```
[안정] Type 계약     → Item.price는 float이다
       Numeric 계약  → total = price × quantity
       State 계약    → PENDING → CONFIRMED → SHIPPED (순서 고정)
[취약] Absence 계약  → cancel_order는 주문을 삭제하지 않는다
```

**부재 계약(Absence Contract)** 은 "하지 말아야 할 것"을 정의하는 계약으로, AST(추상 구문 트리)에 나타나지 않아 코드 분석으로 추출이 어렵고, LLM이 후반 수정 시 위반할 가능성이 가장 높습니다.

---

## 4. 방법론: DRIFT_PROBE 하네스 (방법론적 기여)

계약 드리프트를 측정하기 위해 **DRIFT_PROBE** 테스트를 설계했습니다.

- 각 수정 단계에서 **이전 단계 계약이 여전히 유효한지** 직접 실행 테스트
- 예: step3에서 정의된 "cancel_order는 삭제하지 않는다"를 step4, step6, step7에서 재검증
- 기존 테스트 스위트와 달리 **계약 기원 단계와 검증 단계의 격차**를 추적

이 방법론은 모델에 무관하게 일정한 기준으로 드리프트를 측정합니다.

## 5. 실험 결과 요약

**실험 환경**: Ollama 서버, qwen2.5-coder 모델, OrderSystem Evolution 시나리오 (7단계 누적 수정)

| 모델 | 에이전트 | 최종 단계(step7) 평균 점수 | 비고 |
|---|---|---|---|
| 14b | Baseline | 10.0/10 | 드리프트 없음 |
| 7b | Baseline | 0.0/10 | 5/5 run 완전 붕괴 |
| 7b | SemanticCompressorV2 | 7.2/10 | 4/5 run 생존 |

**주요 관찰**:
1. 14b는 드리프트가 발생하지 않음 — 시나리오가 해당 모델에게 충분히 어렵지 않음
2. 7b의 붕괴는 두 원인이 혼재: ① SyntaxError 등 능력 실패, ② 부재 계약 위반
3. SemanticCompressorV2는 7b에서 유의미한 보존 효과를 보임 (p=0.054, d=1.43)

**한계**: p값이 유의 수준(0.05)에 근접하나 표본이 5회로 제한적. 두 원인(능력 실패 vs 드리프트)의 정량적 분리가 완성되지 않음.
